## Installation & Setup Issues

### Problem: "DLL Not Found" Error

**Error Message:**
```
DllNotFoundException: Unable to load DLL 'spectra': The specified module could not be found
```

**Cause:**
Native library not found at runtime or missing from target platform.

**Solutions:**

**1. Verify Plugin Location**

Check that libraries are in correct paths:
- Windows: `Assets/Spectra/Plugins/Windows/x86_64/spectra.dll` ✓
- macOS: `Assets/Spectra/Plugins/macOS/libspectra.dylib` ✓
- Linux: `Assets/Spectra/Plugins/Linux/x86_64/libspectra.so` ✓

```bash
# Check in Editor
Assets/Spectra/Plugins/
├── Windows/x86_64/spectra.dll          # Check this exists
├── macOS/libspectra.dylib              # Check this exists
└── Linux/x86_64/libspectra.so          # Check this exists
```

**2. Update Plugin Settings**

In Unity Editor:
```
1. Select the DLL file
2. Inspector → Platform Settings
3. Ensure correct platform is selected
4. CPU: x86_64 (or ARM64 for Apple Silicon)
5. Load on startup: ✓ (checked)
```

**3. Rebuild from Source**

If plugins are missing:
```bash
cd native
cmake -B build -DCMAKE_BUILD_TYPE=Release
cmake --build build
# Copy generated libraries to Assets/Spectra/Plugins/
```

**4. Check Platform**

```csharp
// Verify which platform runtime detected
Debug.Log($"Running on: {Application.platform}");
```

### Problem: P/Invoke Signature Mismatch

**Error Message:**
```
EntryPointNotFoundException: Unable to find entry point named 'spectra_butter_bp' in DLL 'spectra'
```

**Cause:**
- Native library compiled with different version
- Function name mangling issue
- Library corruption

**Solutions:**

**1. Verify Library Version**

```bash
# Check symbols in library
# Linux/macOS:
nm -D libspectra.so | grep spectra_butter

# Windows:
dumpbin /exports spectra.dll | findstr spectra_butter
```

**2. Ensure C ABI Export**

Native library must export C symbols (not C++ mangled):

```cpp
// Correct (in spectra_abi.cpp):
extern "C" {
    int spectra_butter_bp(int order, float low_freq, float high_freq,
                         int type, float* b, float* a,
                         int* b_len, int* a_len)
    {
        // Implementation
    }
}

// Wrong (without extern "C"):
int spectra_butter_bp(...)  // Will be name-mangled!
```

**3. Recompile Native Library**

```bash
cd native
rm -rf build
cmake -B build -DCMAKE_BUILD_TYPE=Release
cmake --build build
```

## Filter Design Problems

### Problem: Unstable Filter (NaN Output)

**Symptoms:**
```csharp
float[] filtered = DSP.LFilter(b, a, signal);
// Result contains NaN or Infinity values
Debug.Log(filtered[0]);  // NaN
```

**Causes:**
1. Frequency range too extreme (0.001-0.999)
2. Filter order too high (> 12)
3. Numerically unstable coefficient values

**Solutions:**

**1. Check Frequency Range**

```csharp
// Problem: Extreme frequencies
var (b, a) = DSP.Butter(4, 0.001f, 0.999f, FilterType.Bandpass);

// Solution: Use more reasonable range
var (b, a) = DSP.Butter(4, 0.1f, 0.9f, FilterType.Bandpass);
```

**Recommended ranges:**
- Lowpass: 0.1-0.8 (normalized)
- Highpass: 0.1-0.8
- Bandpass: low < center < high, all 0.1-0.8
- Bandstop: low < center < high, all 0.1-0.8

**2. Reduce Filter Order**

```csharp
// Problem: Order too high
var (b, a) = DSP.Butter(10, 0.2f, 0.5f, FilterType.Bandpass);

// Solution: Reduce order
var (b, a) = DSP.Butter(4, 0.2f, 0.5f, FilterType.Bandpass);
```

**Typical order selection:**
- Audio general purpose: 2-4
- Speech enhancement: 4-6
- Hum removal: 6-8
- Sharp transitions: 2-4 (higher orders risk instability)

**3. Validate Filter Before Use**

```csharp
public static bool ValidateFilter(float[] b, float[] a, float[] signal)
{
    // Check coefficients
    foreach (float coeff in b)
        if (float.IsNaN(coeff) || float.IsInfinity(coeff))
            return false;

    foreach (float coeff in a)
        if (float.IsNaN(coeff) || float.IsInfinity(coeff))
            return false;

    // Test filter on small signal
    float[] test = new float[Math.Min(1024, signal.Length)];
    Array.Copy(signal, test, test.Length);

    float[] result = DSP.LFilter(b, a, test);

    foreach (float val in result)
        if (float.IsNaN(val) || float.IsInfinity(val))
            return false;

    return true;
}

// Usage:
if (!ValidateFilter(b, a, signal))
{
    Debug.LogError("Filter is unstable!");
    return;
}
```

### Problem: Filter Has No Effect

**Symptoms:**
```csharp
float[] filtered = DSP.LFilter(b, a, signal);
// Filtered sound identical to original
```

**Causes:**
1. Wrong frequency normalization
2. Coefficients represent all-pass filter
3. Filter not applied to signal
4. Filter type reversed (BP instead of BS)

**Solutions:**

**1. Verify Frequency Normalization**

```csharp
// WRONG: Frequencies not normalized
float targetFreq = 1000f;  // Hz
var (b, a) = DSP.Butter(4, targetFreq, targetFreq*2, FilterType.Bandpass);

// CORRECT: Normalize to Nyquist
float sampleRate = AudioSettings.outputSampleRate;  // 44100 Hz
float normalizedLow = 1000f / (sampleRate / 2f);    // ≈ 0.0454
float normalizedHigh = 2000f / (sampleRate / 2f);   // ≈ 0.0907
var (b, a) = DSP.Butter(4, normalizedLow, normalizedHigh, FilterType.Bandpass);
```

**Quick check:**
```csharp
// Normalized frequencies should be between 0 and 1
Debug.Assert(normalizedFreq > 0 && normalizedFreq < 1,
    "Frequency must be normalized between 0 and 1");
```

**2. Test with Known Tone**

```csharp
public static void TestFilterWithTone()
{
    // Generate 1 kHz test tone
    float[] testSignal = new float[44100];
    for (int i = 0; i < testSignal.Length; i++)
        testSignal[i] = Mathf.Sin(2f * Mathf.PI * 1000f * i / 44100f) * 0.5f;

    // Design bandpass filter around 1 kHz
    var (b, a) = DSP.Butter(4, 0.02f, 0.05f, FilterType.Bandpass);

    // Apply filter
    float[] filtered = DSP.LFilter(b, a, testSignal);

    // Check power before and after
    float inputPower = CalculatePower(testSignal);
    float outputPower = CalculatePower(filtered);

    Debug.Log($"Input power: {inputPower:F3}");
    Debug.Log($"Output power: {outputPower:F3}");
    Debug.Log($"Gain: {20 * Mathf.Log10(outputPower / inputPower):F2} dB");

    // Bandpass should show gain > -3dB at center frequency
    // Bandstop should show gain < -20dB
}

static float CalculatePower(float[] signal)
{
    float sum = 0;
    foreach (float sample in signal)
        sum += sample * sample;
    return sum / signal.Length;
}
```

**3. Check Filter Type**

```csharp
// Make sure you're using correct type:
// Bandpass: Passes frequencies BETWEEN low and high ✓
// Bandstop: BLOCKS frequencies between low and high ✗

// For speech enhancement (want to keep 60-3500 Hz):
var (b, a) = DSP.Butter(4, 0.003f, 0.159f, FilterType.Bandpass);  // Correct!

// NOT:
var (b, a) = DSP.Butter(4, 0.003f, 0.159f, FilterType.Bandstop);  // Wrong!
```

## Audio Processing Issues

### Problem: Audio Distortion/Clipping

**Symptoms:**
```
Filtered audio sounds crackling, distorted, or extremely quiet/loud
```

**Causes:**
1. Signal overflow/underflow during filtering
2. Gain mismatch (filter attenuates too much)
3. Floating-point precision loss

**Solutions:**

**1. Normalize Input Signal**

```csharp
public static float[] NormalizeSignal(float[] signal, float targetMax = 0.5f)
{
    // Find peak
    float peak = 0;
    foreach (float sample in signal)
        peak = Mathf.Max(peak, Mathf.Abs(sample));

    if (peak == 0) return signal;

    // Normalize to target
    float[] normalized = new float[signal.Length];
    float scale = targetMax / peak;

    for (int i = 0; i < signal.Length; i++)
        normalized[i] = signal[i] * scale;

    return normalized;
}

// Usage:
float[] normalized = NormalizeSignal(audioData, 0.4f);
float[] filtered = DSP.LFilter(b, a, normalized);
```

**2. Measure Filter Gain**

```csharp
public static float MeasureFilterGain(float[] b, float[] a)
{
    // Generate test tone
    float[] testTone = new float[1024];
    for (int i = 0; i < testTone.Length; i++)
        testTone[i] = Mathf.Sin(2f * Mathf.PI * 0.25f * i);  // 0.25 normalized freq

    // Apply filter
    float[] filtered = DSP.LFilter(b, a, testTone);

    // Measure power
    float inputPower = CalculatePower(testTone);
    float outputPower = CalculatePower(filtered);

    return outputPower / inputPower;
}

// Usage:
float gain = MeasureFilterGain(b, a);
Debug.Log($"Filter gain: {gain:F3} ({20*Mathf.Log10(gain):F2} dB)");
```

**3. Apply Compensating Gain**

```csharp
public class GainCompensatedFilter : MonoBehaviour
{
    private StreamingIIRFilter filter;
    private float compensatingGain = 1.0f;

    void Start()
    {
        var (b, a) = DSP.Butter(4, 0.2f, 0.5f, FilterType.Bandpass);
        filter = new StreamingIIRFilter(b, a);

        // Measure gain at center frequency
        float centerGain = MeasureFilterGain(b, a);
        compensatingGain = 1.0f / centerGain;  // Invert it

        Debug.Log($"Applying compensating gain: {compensatingGain:F3}");
    }

    void OnAudioFilterRead(float[] data, int channels)
    {
        float[] mono = ConvertToMono(data, channels);
        float[] filtered = filter.Process(mono);

        // Apply compensating gain
        for (int i = 0; i < filtered.Length; i++)
            filtered[i] *= compensatingGain;

        ConvertToStereo(filtered, data);
    }
}
```

### Problem: Audio Latency

**Symptoms:**
```
Filtered audio delayed compared to original
Visual-audio sync issues in game
```

**Causes:**
1. FiltFilt (zero-phase) adds 2× latency
2. Large buffer sizes in AudioSettings
3. Multiple cascaded filters

**Solutions:**

**1. Choose Appropriate Filtering Method**

```csharp
// Real-time playback: Use LFilter (zero latency)
float[] filtered = DSP.LFilter(b, a, inputData);

// Post-processing: Use FiltFilt (2× buffer latency, but zero phase)
float[] filtered = DSP.FiltFilt(b, a, inputData);
```

**Latency comparison:**
- **LFilter**: Zero added latency, but phase distortion
- **FiltFilt**: 2× buffer latency (23 ms @ 1024 samples, 44.1 kHz), zero phase distortion

**2. Minimize Buffer Size**

```csharp
// In AudioClip streaming:
// Smaller buffer = less latency but more CPU overhead

// Typical settings:
// Gaming: 512-1024 samples (~12-23 ms)
// Real-time processing: 256-512 samples (~6-12 ms)
// Post-processing: 2048+ samples (~46+ ms, latency doesn't matter)
```

**3. Measure Actual Latency**

```csharp
public static float MeasureFilterLatency(float[] b, float[] a, int bufferSize)
{
    // Generate impulse
    float[] impulse = new float[bufferSize * 4];
    impulse[0] = 1.0f;

    // Apply filter
    float[] response = DSP.LFilter(b, a, impulse);

    // Find peak
    int peakIndex = 0;
    float peakMag = 0;
    for (int i = 0; i < response.Length; i++)
    {
        if (Mathf.Abs(response[i]) > peakMag)
        {
            peakMag = Mathf.Abs(response[i]);
            peakIndex = i;
        }
    }

    float sampleRate = AudioSettings.outputSampleRate;
    float latencySeconds = peakIndex / (float)sampleRate;

    return latencySeconds;
}

// Usage:
float latency = MeasureFilterLatency(b, a, 1024);
Debug.Log($"Filter latency: {latency*1000:F2} ms");
```

## Performance Problems

### Problem: High CPU Usage

**Symptoms:**
```
Frame rate drops when filter is active
Audio stuttering/glitching
```

**Causes:**
1. Filter order too high
2. Multiple high-order filters running
3. Filter redesign in audio thread
4. Memory allocations in hot path

**Solutions:**

**1. Profile Filter Performance**

```csharp
public static void ProfileFilterPerformance(float[] b, float[] a)
{
    int iterations = 100;
    int bufferSize = 1024;

    float[] testSignal = new float[bufferSize];
    for (int i = 0; i < testSignal.Length; i++)
        testSignal[i] = UnityEngine.Random.Range(-1f, 1f);

    var sw = System.Diagnostics.Stopwatch.StartNew();

    for (int i = 0; i < iterations; i++)
    {
        DSP.LFilter(b, a, testSignal);
    }

    sw.Stop();

    float msPerBuffer = sw.ElapsedMilliseconds / (float)iterations;
    float percentOfFrame = (msPerBuffer / 16.67f) * 100f;  // At 60 FPS

    Debug.Log($"Time per buffer: {msPerBuffer:F2} ms ({percentOfFrame:F1}% of frame budget)");
}
```

**2. Use StreamingIIRFilter (Reusable)**

```csharp
// BAD: Creates new filter every frame
void OnAudioFilterRead(float[] data, int channels)
{
    var (b, a) = DSP.Butter(4, 0.2f, 0.5f, FilterType.Bandpass);  // Expensive!
    float[] filtered = DSP.LFilter(b, a, data);
}

// GOOD: Reuse filter
private StreamingIIRFilter filter;

void Start()
{
    var (b, a) = DSP.Butter(4, 0.2f, 0.5f, FilterType.Bandpass);
    filter = new StreamingIIRFilter(b, a);
}

void OnAudioFilterRead(float[] data, int channels)
{
    float[] filtered = filter.Process(data);  // Fast!
}
```

**3. Reduce Filter Order**

```csharp
// Too expensive
var (b, a) = DSP.Butter(10, 0.2f, 0.5f, FilterType.Bandpass);

// More affordable
var (b, a) = DSP.Butter(4, 0.2f, 0.5f, FilterType.Bandpass);

// CPU difference: ~3.5× speedup
```

**4. Cache Filter Coefficients**

```csharp
private Dictionary<string, (float[], float[])> coefficientCache = new();

(float[], float[]) GetOrCreateFilter(string key, System.Func<(float[], float[])> factory)
{
    if (!coefficientCache.TryGetValue(key, out var coeffs))
    {
        coeffs = factory();
        coefficientCache[key] = coeffs;
    }
    return coeffs;
}

// Usage:
var (b, a) = GetOrCreateFilter("speech_bandpass",
    () => DSP.Butter(4, 0.003f, 0.159f, FilterType.Bandpass));
```

## Compatibility Issues

### Problem: Different Results on Different Platforms

**Symptoms:**
```
Audio sounds different on Windows vs Mac vs Linux
Different filter response across platforms
```

**Causes:**
1. Floating-point precision differences
2. Different SIMD implementations
3. Compiler optimizations

**Solutions:**

**1. Use Double Precision for Calculations**

```csharp
// Instead of float calculations
float[] doubleFilter(float[] b, float[] a, float[] signal)
{
    // Convert to double
    double[] b_d = System.Array.ConvertAll(b, x => (double)x);
    double[] a_d = System.Array.ConvertAll(a, x => (double)x);
    double[] sig_d = System.Array.ConvertAll(signal, x => (double)x);

    // Process in double precision
    double[] output_d = new double[sig_d.Length];
    // ... filter in double ...

    // Convert back to float
    return System.Array.ConvertAll(output_d, x => (float)x);
}
```

**2. Normalize Results Across Platforms**

```csharp
public static float NormalizeCrossPlatform(float value)
{
    // Clamp to prevent extreme values
    return Mathf.Clamp(value, -1f, 1f);
}
```

## Frequently Asked Questions

### Q1: Should I use Bandpass or Bandstop for My Application?

**Bandpass:**
- Keep frequencies WITHIN a range
- Use for: Speech enhancement, acoustic analysis, music frequency bands
- Example: Keep 60-3500 Hz, remove everything else

**Bandstop:**
- Remove frequencies WITHIN a range
- Use for: Hum removal, artifact removal
- Example: Remove 60 Hz hum, keep everything else

**Quick decision:**
- "I want to KEEP specific frequencies" → Bandpass
- "I want to REMOVE specific frequencies" → Bandstop

---

### Q2: What Order Should I Use?

**General Guidelines:**
```
Audio Quality Purpose:
  - Order 2: Gentle, transparent filtering
  - Order 4: Moderate, noticeable effect
  - Order 6+: Aggressive, potential artifacts

Stability Concern:
  - Order ≤ 4: Very stable, safe for most uses
  - Order 4-6: Stable, verify coefficients
  - Order > 6: Verify stability before deployment

Hum Removal:
  - Order 6-8: Narrow, effective notch
  - Order 10+: Very narrow, risk instability

General Recommendation: Start with order 4, adjust as needed
```

### Q3: Can I Change Filter Frequency in Real-Time?

**Yes, but with caveats:**

```csharp
// Not efficient: Redesign every frame
void Update()
{
    var (b, a) = DSP.Butter(4, targetFreq, targetFreq+0.2f, FilterType.Bandpass);
    // Expensive every frame!
}

// Better: Smooth parameter changes
private float currentFreq;
private float targetFreq;

void Update()
{
    currentFreq = Mathf.Lerp(currentFreq, targetFreq, 0.05f);

    if (Mathf.Abs(currentFreq - lastDesignedFreq) > 0.01f)
    {
        var (b, a) = DSP.Butter(4, currentFreq, currentFreq+0.2f, FilterType.Bandpass);
        filter = new StreamingIIRFilter(b, a);
        lastDesignedFreq = currentFreq;
    }
}
```

### Q4: How Do I Achieve Zero-Phase Filtering?

Use `FiltFilt` instead of `LFilter`:

```csharp
// Standard filtering (has phase shift)
float[] filtered1 = DSP.LFilter(b, a, signal);

// Zero-phase filtering (no phase shift, but 2× latency)
float[] filtered2 = DSP.FiltFilt(b, a, signal);
```

### Q6: Can I Use Multiple Filters on Same Audio?

Yes, cascade them:

```csharp
public class MultiStageFilter : MonoBehaviour
{
    private StreamingIIRFilter[] filters;

    void Start()
    {
        filters = new StreamingIIRFilter[3];

        // Speech enhancement: Remove rumble, then keep voice, then remove hiss
        var (b1, a1) = DSP.Butter(2, 0.001f, 0.002f, FilterType.Bandstop);  // < 20 Hz
        var (b2, a2) = DSP.Butter(4, 0.003f, 0.159f, FilterType.Bandpass); // 60-3500 Hz
        var (b3, a3) = DSP.Butter(2, 0.18f, 0.25f, FilterType.Bandstop);   // > 5 kHz

        filters[0] = new StreamingIIRFilter(b1, a1);
        filters[1] = new StreamingIIRFilter(b2, a2);
        filters[2] = new StreamingIIRFilter(b3, a3);
    }

    void OnAudioFilterRead(float[] data, int channels)
    {
        float[] processed = ConvertToMono(data, channels);

        // Apply cascade
        foreach (var filter in filters)
            processed = filter.Process(processed);

        ConvertToStereo(processed, data);
    }
}
```

### Q7: How Do I Export Filter Coefficients?

Use `DSP.Freqz` or direct coefficient access:

```csharp
public static void ExportFilter(float[] b, float[] a, string filename)
{
    StringBuilder csv = new StringBuilder();

    // Header
    csv.AppendLine("Type,Index,Coefficient");

    // Coefficients
    for (int i = 0; i < b.Length; i++)
        csv.AppendLine($"b,{i},{b[i]:F16}");

    for (int i = 0; i < a.Length; i++)
        csv.AppendLine($"a,{i},{a[i]:F16}");

    System.IO.File.WriteAllText(filename, csv.ToString());
    Debug.Log($"Exported to {filename}");
}

// Usage:
var (b, a) = DSP.Butter(4, 0.2f, 0.5f, FilterType.Bandpass);
ExportFilter(b, a, "bandpass_filter.csv");
```
