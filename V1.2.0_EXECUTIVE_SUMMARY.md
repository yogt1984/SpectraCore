# SpectraCore v1.2.0 - Executive Summary

**Date:** February 26, 2026
**Status:** Planning Phase - Decision Required

---

## 🎯 Quick Overview

v1.2.0 focuses on **platform expansion** and **enhanced filter capabilities**:

- **8 Platforms Total:** Linux ✅ | Windows ✅ | macOS ✅ | iOS 🎯 | Android 🎯 | WebGL 🎯
- **6 Filter Families:** Butterworth, Chebyshev I/II, Elliptic ✅ | Bessel 🎯 | Parks-McClellan 🎯
- **Timeline:** 9 weeks (~2 months)
- **Target Release:** End of April 2026

---

## 📊 Recommended Scope (Prioritized)

### ✅ INCLUDE in v1.2.0

| Feature | Priority | Effort | User Impact |
|---------|----------|--------|-------------|
| **iOS & Android** | P0 Must-Have | 2-3 weeks | 🔥 Critical - Mobile games |
| **WebGL/WASM** | P0 Must-Have | 2 weeks | 🔥 Critical - Web deployment |
| **Bessel Filters** | P1 Should-Have | 1 week | 🟡 Medium - Linear phase apps |
| **Parks-McClellan FIR** | P1 Should-Have | 2 weeks | 🟡 Medium - Optimal FIR design |
| **SIMD Foundation** | P1 Should-Have | 1.5 weeks | 🟡 Medium - 2-4× FIR speedup |

**Total:** ~9 weeks for complete v1.2.0

### ⏸️ DEFER to v1.3.0

- MFCC and spectral features (P2 - niche use cases)
- Full Burst integration (P1 - complex, needs v1.2 stability first)
- Preset system (P2 - UX enhancement)
- Real-time visualization (P3 - not critical)

## 🚀 Implementation Phases

```
Week 1-3:  iOS + Android builds, testing
Week 2-4:  WebGL/WASM build, browser testing
Week 4-5:  Bessel filters (all response types)
Week 4-6:  Parks-McClellan FIR (Remez algorithm)
Week 5-7:  SIMD optimization (foundation)
Week 7-9:  Cross-platform integration, release prep
```

**Parallelization:** iOS/Android can run parallel with WebGL (different platforms)

- **v1.2.0 (6 weeks):** Mobile + WebGL + Bessel
